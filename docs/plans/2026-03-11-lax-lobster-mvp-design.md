# 龙虾海底跑酷 MVP 设计摘要

## 设计输入

- 来源：`_bmad-output/implementation-artifacts/tech-spec-lax-lobster-mvp-tech-spec.md`
- 代码库状态：全新项目，无既有源码
- 目标：用 `Phaser 3 + TypeScript + Vite` 落地一个可玩的 Web MVP

## 设计结论

### 核心循环

- 玩家通过按住/松开控制龙虾下潜与上浮。
- 场景持续向右滚动，通过程序化生成障碍、捕食者、道具维持节奏。
- 得分驱动难度提升与进化解锁。
- 死亡后优先走免费复活，否则进入结算。

### 架构拆分

- `config/`：所有策划数值配置化，避免魔法数字散落。
- `data/`：负责运行态状态、玩家持久化与降级存储。
- `systems/`：输入、滚动、难度、碰撞、进化、对象池、音效、粒子。
- `entities/`：龙虾、障碍、捕食者、道具，尽量保持职责单一。
- `scenes/`：引导、菜单、游戏、结算、进化切换。
- `ui/`：HUD、复活面板、进化卡片。

### MVP 取舍

- 保留：单指/单键输入、一局完整流程、进化切换、免费复活、护盾、捕食者、LocalStorage、基础粒子与音效。
- 延后：广告复活、金币复活、驱鱼剂、回溯珍珠、外部美术资源、复杂商业化系统。

### 测试策略

- 自动化：优先覆盖 `StorageManager`、`GameState`、`DifficultyManager`、`PlayerData`。
- 手动：通过 `tests/gameplay-checklist.md` 覆盖手感、边界保护、复活、进化解锁与双端一致性。

## 风险与约束

- 项目当前不是 Git 仓库，无法基于提交做增量审查。
- Phaser 视觉与交互强相关，部分行为更适合手动验证而非纯单测。
- 规格很大，MVP 需要优先保证核心验收链路可玩且稳定。
